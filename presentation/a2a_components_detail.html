A2A 구성 요소 상세

Agent-to-Agent Protocol의 5가지 핵심 구성 요소

1

Agent Card

에이전트의 프로필 카드

에이전트의 신원, 능력, 연락처 정보를 담은 명함과 같은 역할. 다른 에이전트가 이 카드를 보고 어떤 작업을 요청할 수 있는지 판단합니다.

  • 에이전트 이름 및 설명
  • 제공 가능한 서비스 목록
  • 통신 엔드포인트 정보
  • 지원 프로토콜 버전

예시

{
  "name": "SentimentAnalysisAgent",
  "capabilities": ["analyze_sentiment", "extract_keywords"],
  "endpoint": "http://localhost:8202"
}
2

Task

작업 요청 단위

한 에이전트가 다른 에이전트에게 요청하는 구체적인 작업 단위. 작업의 유형, 입력 데이터, 예상 출력 형식 등을 정의합니다.

  • 작업 ID 및 타입
  • 입력 파라미터 정의
  • 우선순위 및 타임아웃
  • 예상 결과 형식

예시

{
  "task_id": "analyze-001",
  "type": "sentiment_analysis",
  "input": {"text": "긍정적인 수익률..."},
  "priority": "high"
}
3

Artifact

작업 결과물 및 중간 산출물

에이전트가 생성하거나 처리한 데이터, 파일, 분석 결과 등의 산출물. 다른 에이전트와 공유 가능한 형태로 패키징되어 전달됩니다.

  • 분석 리포트, 차트, 그래프
  • 처리된 데이터셋
  • 생성된 코드나 설정 파일
  • 메타데이터 및 버전 정보

예시

{
  "artifact_id": "report-20250722",
  "type": "sentiment_report",
  "format": "markdown",
  "content": "## 투자 심리 분석 보고서\n### 전반적 심리: 긍정적 (78%)...",
  "metadata": {"created_at": "2025-07-22T10:30:00Z", "confidence": 0.92}
}
4

Message

에이전트 간 통신 메시지

에이전트 간 주고받는 모든 통신의 기본 단위. 요청, 응답, 이벤트, 에러 등 다양한 타입의 정보를 전달합니다.

  • 메시지 타입 (REQUEST/RESPONSE/EVENT)
  • 송신자/수신자 정보
  • 타임스탬프 및 추적 ID
  • 페이로드 및 헤더

예시

{
  "msg_type": "REQUEST",
  "from": "orchestrator",
  "to": "news_agent",
  "payload": {"action": "fetch_news"}
}
5

Part

대용량 데이터 분할 전송

큰 데이터나 스트리밍 데이터를 여러 부분으로 나누어 전송하는 단위. 순차적 처리와 병렬 처리를 모두 지원합니다.

  • 청크 번호 및 전체 크기
  • 스트리밍 지원
  • 체크섬 및 무결성 검증
  • 재전송 메커니즘

예시

{
  "part_number": 3,
  "total_parts": 10,
  "data": "[부분 데이터...]",
  "checksum": "sha256:abcd..."
}
💡 이 5가지 구성 요소는 A2A 프로토콜의 핵심으로, 서로 다른 프레임워크로 만들어진 AI 에이전트들이 표준화된 방식으로 협업할 수 있게 합니다.

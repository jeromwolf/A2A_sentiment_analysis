Agent-to-Agent 프로토콜 등장 배경

01

MCP 이후

MCP: '에이전트-도구' 연결
→ 수직적 통합 문제를 해결
강력해진 개별 에이전트
But, 여전히 고립된 전문가로 존재
각기 다른 회사나 개발자가 만든 전문화된 에이전트들 간 협력 방법 부재
02

이기종 프레임워크

진정한 의미의 분산형
자율 시스템 구축에 한계
에이전트 간 서로 협업할 공통 통신 언어 (프로토콜)가 없음
예: 여행 지원 AI : 항공권 예약 에이전트가 저렴한 항공편을 찾았지만, 숙소 검색 에이전트는 비행기 도착 시간과 맞지 않는 호텔 추천
03

A2A 의 필요성

자율적인 워크플로우 구축을 위해 에이전트 간 수평적 통합 표준 필요
수직적 통합 (with MCP)
: 에이전트가 자신의 하위 도구/자원과 통신
수평적 통합 (with A2A)
: 에이전트가 다른 에이전트와 통신

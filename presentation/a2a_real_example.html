A2A 실제 구현 예시 : SEC Agent

SEC(미국 증권거래위원회) 공시 데이터를 수집하는 에이전트의 A2A 구현

AgentCard - SEC Agent 프로필

SEC Agent가 Service Registry에 등록하는 자신의 프로필

{
  "agent_id": "sec-agent-v2",
  "name": "SEC Agent V2",
  "endpoint": "http://localhost:8210",
  "capabilities": [
    {
      "name": "sec_data_collection",
      "description": "SEC 공시 수집"
    }
  ]
}

Task - 데이터 수집 작업

Orchestrator가 SEC Agent에게 할당하는 작업

Task ID: task-123456
Action: "collect_sec_data"
Ticker: "AAPL"
Status: "in_progress"
Priority: "HIGH"

Message - A2A 메시지

Orchestrator → SEC Agent 메시지

{
  "header": {
    "sender_id": "orchestrator-v2",
    "receiver_id": "sec-agent-v2",
    "message_type": "REQUEST"
  },
  "body": {
    "action": "collect_sec_data",
    "payload": { "ticker": "AAPL" }
  }
}

Artifact - 수집된 공시 데이터

SEC Agent가 생성한 결과물

[
  {
    "form_type": "10-K",
    "filed_date": "2024-10-30",
    "title": "Annual Report",
    "url": "https://..."
  },
  {
    "form_type": "8-K",
    "title": "Current Report"
  }
]

SEC Agent A2A 통신 흐름

👤 1. Agent 등록

SEC Agent → Registry Server

AgentCard를 통해 자신의 능력을 등록

POST /register

↓

✉️ 2. 작업 요청 수신

Orchestrator → SEC Agent

A2A Message로 AAPL 공시 수집 요청

MessageType: REQUEST

↓

📋 3. Task 수행

SEC Agent 내부 처리

• EDGAR API 호출

• 10-K, 10-Q, 8-K 문서 수집

• 데이터 파싱 및 정리

↓

📦 4. Artifact 전송

SEC Agent → Orchestrator

수집된 공시 데이터를 A2A Message로 전송

MessageType: RESPONSE

결과: 5개 SEC 공시 문서

🚀 실시간 A2A 통신 시뮬레이션

SEC Agent가 실제로 A2A 프로토콜을 사용하여 통신하는 과정을 확인해보세요

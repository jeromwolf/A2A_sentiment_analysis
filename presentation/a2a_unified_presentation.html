A2A의 핵심: 기능의 서비스화

• A2A (Agent-to-Agent) 의 작동 원리

: 각 에이전트가 자신의 핵심 기능을 '서비스'로 외부에 제공

• Agent 가 제공하는 서비스로서의 기능 (Capabilities as a Service)

예: 이메일 요약 에이전트의 '긴 글 요약하기' 기능
예: 데이터베이스 에이전트의 '특정 조건으로 고객 데이터 조회하기' 기능
예: UI 디자인 에이전트의 '주어진 내용으로 UI 컴포넌트 생성하기' 기능

• 마이크로서비스 아키텍처(MSA)와의 개념의 유사성

: A2A는 AI 에이전트 세계에 마이크로서비스 개념을 도입.

요청하는 에이전트는 다른 에이전트가 어떻게(How) 그 일을 하는지 알 필요 없이,
무엇(What)을 할 수 있는지 알고 표준화된 방식으로 요청만 하면 됨

기능서비스화 마이크로서비스 MSA 에이전트API

A2A의 핵심 비유: AI 에이전트들의 소셜 네트워크

🔨

MCP

• 에이전트와 다양한 외부 자원과 맥락(Context) 정보를

일관된 형태로 주고받을 수 있도록 정의한 표준 규약

+
🤝

A2A

• 에이전트와 에이전트를 연결하여 네트워크를 형성하고

이기종 에이전트 간 공통된 형식으로 통신할 수 있는 표준 규약

• AI 에이전트들의 SNS 라고 생각한다면:

프로필 (Agent Card) 을 공개하고,
친구를 맺으며 (Agent Discovery)
메시지를 보내서 요청하고 (Task / Message)
상호작용하며 협력 기반을 제공 (Artifact)

= SYNERGY

수직적 통합(MCP) + 수평적 통합(A2A) = 차세대 AI 시스템

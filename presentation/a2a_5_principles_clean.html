Agent-to-Agent Protocol

설계 5대 원칙

1

에이전트 능력 수용 (Embrace Agentic Capabilities)

에이전트들을 단순한 "도구"로 제한하지 않고 자율적 주체로 인정. 메모리, 도구 사용 등을 명시적으로 노출하지 않아도 자연스러운 협업 가능. 다양한 에이전트 임원들이 수 있도록 지원하는 데 초점

2

기존 웹 표준 활용 (Build on Web Standards)

HTTP/HTTPS, JSON-RPC 등 검증된 웹 표준 기반. 새로운 프로토콜을 만들지 않고 기존 인프라 활용

HTTP/2 JSON-RPC WebSocket
3

기본적 보안을 확보 (Secure by Default)

OAuth 2.0, JWT 등 표준 인증 메커니즘 지원. 엔터프라이즈급 보안 요구사항 충족

OAuth 2.0 JWT TLS 1.3
4

장기 실행 작업 지원 (Support Long-Running Tasks)

수 시간 또는 수일이 걸리는 작업 지원. 인간 개입이 필요한 워크플로우 처리 가능. 실시간 피드백, 일시 정지, 상태 업데이트 제공

5

모달리티 무관 (Modality Agnostic)

텍스트, 이미지, 오디오, 파일 등 다양한 데이터 형식 지원. 메시지 형식에 구애받지 않는 통신

A2A Protocol - Google이 2025년 발표한 차세대 AI 에이전트 통신 표준

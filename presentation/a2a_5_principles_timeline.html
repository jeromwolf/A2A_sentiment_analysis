Agent-to-Agent Protocol

설계 5대 원칙

1

에이전트 능력 수용

에이전트를 단순한 "도구"로 제한하지 않고 자율적 주체로 인정. 메모리, 도구 사용 등을 명시적으로 노출하지 않아도 자연스러운 협업 가능

2

기존 웹 표준 활용

검증된 웹 표준 기반으로 새로운 프로토콜을 만들지 않고 기존 인프라 활용

HTTP/HTTPS JSON-RPC WebSocket
3

기본적 보안 확보

표준 인증 메커니즘 지원으로 엔터프라이즈급 보안 요구사항 충족

OAuth 2.0 JWT TLS 1.3
4

장기 실행 작업 지원

수 시간 또는 수일이 걸리는 작업 지원. 인간 개입이 필요한 워크플로우 처리. 실시간 피드백, 일시 정지, 상태 업데이트 제공

5

모달리티 독립

다양한 데이터 형식 지원으로 메시지 형식에 구애받지 않는 통신

텍스트 이미지 오디오 파일
